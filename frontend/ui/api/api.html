Developer Access

Environmental data and alert access for cities

Use these endpoints to retrieve aggregated operator data, read public alerts, and subscribe external systems to public alert notifications.

GET /api/operator?cityId=hamilton&minutes=60

City Aggregated Metrics

Returns aggregated city/operator data for the selected time window. Use this for dashboards, zone summaries, and sensor metric rollups.

Example response fields

{
  "cityId": "hamilton",
  "cityName": "Hamilton",
  "zones": [...],
  "metrics": [...],
  "generatedAt": "2026-04-06T15:00:00.000Z"
}
GET /api/alertsystem/public-alerts?cityId=hamilton

Public Alerts

Returns public-facing active alerts only. Use this for public websites, partner systems, mobile apps, and third-party dashboards.

Example response fields

{
  "alerts": [
    {
      "alertId": "alert-123",
      "cityId": "hamilton",
      "zoneId": "central-hamilton",
      "sensorType": "airQuality",
      "severity": "severe",
      "message": "Poor air quality detected."
    }
  ]
}
POST /api/subscriptions

Subscribe to Public Alerts

Register a subscriber for public alert notifications. This is the missing backend piece you still need to add in AWS.

Example request body

{
  "cityId": "hamilton",
  "channel": "webhook",
  "target": "https://example.com/webhooks/geoalert"
}
POST Alert notification payload

Payload sent to subscribers

When a public alert becomes active, GeoAlert should POST a payload like this to each subscriber webhook for that city.

Example notification payload

{
  "event": "public-alert-created",
  "sentAt": "2026-04-06T15:05:00.000Z",
  "alert": {
    "alertId": "alert-123",
    "cityId": "hamilton",
    "zoneId": "central-hamilton",
    "zoneName": "Central Hamilton",
    "sensorType": "airQuality",
    "severity": "severe",
    "message": "Poor air quality detected.",
    "status": "active",
    "isPublic": true
  }
}
