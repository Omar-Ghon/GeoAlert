Phone Alerts API Documentation

Overview

The Phone Alerts API allows you to send SMS notifications to users when sensor readings exceed configured thresholds. This is part of the GeoAlert system for real-time environmental monitoring.

Authentication

Include your API key in the request header:

Authorization: Bearer YOUR_API_KEY

Endpoints

POST /api/alerts/send-sms

Send an SMS alert to a user's phone number

Request Body

{
  "phoneNumber": "+1-555-123-4567",
  "message": "[GeoAlert WARNING] Hamilton: Air Quality at 150AQI. Check dashboard for details.",
  "priority": "high",
  "timestamp": "2026-04-01T10:30:00Z"
}
Field Type Required Description
phoneNumber string Yes User's phone number in E.164 format or common formats
message string Yes SMS message (max 160 characters recommended)
priority string No Alert priority: 'critical' or 'high' (default: 'high')
timestamp string No ISO 8601 timestamp of when alert was triggered

Response

{
  "success": true,
  "messageId": "msg_123456789",
  "phoneNumber": "+1-555-123-4567",
  "sentAt": "2026-04-01T10:30:00Z",
  "status": "delivered"
}
Field Type Description
success boolean Whether the SMS was sent successfully
messageId string Unique identifier for tracking the message
phoneNumber string The phone number the SMS was sent to
sentAt string ISO 8601 timestamp of when the SMS was sent
status string Delivery status: 'pending', 'delivered', 'failed'

Error Response

{
  "success": false,
  "error": "Invalid phone number format",
  "code": "INVALID_PHONE"
}
Error Code Status Code Description
INVALID_PHONE 400 Phone number format is invalid
MESSAGE_TOO_LONG 400 Message exceeds maximum length (320 characters)
QUOTA_EXCEEDED 429 Rate limit exceeded (max 100 SMS per hour)
SMS_SERVICE_ERROR 503 SMS provider is temporarily unavailable

Usage Examples

JavaScript (Node.js)

const axios = require('axios');

const sendAlert = async (phoneNumber, message) => {
  try {
    const response = await axios.post(
      'https://api.geoalert.com/api/alerts/send-sms',
      {
        phoneNumber: phoneNumber,
        message: message,
        priority: 'high'
      },
      {
        headers: {
          'Authorization': 'Bearer YOUR_API_KEY',
          'Content-Type': 'application/json'
        }
      }
    );

    console.log('Alert sent:', response.data);
    return response.data;
  } catch (error) {
    console.error('Error sending alert:', error.response.data);
  }
};

// Send alert
sendAlert('+1-555-123-4567', '[GeoAlert WARNING] Hamilton: Air Quality at 155AQI.');

Python

import requests
import json

def send_alert(phone_number, message):
    url = 'https://api.geoalert.com/api/alerts/send-sms'
    
    headers = {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    }
    
    payload = {
        'phoneNumber': phone_number,
        'message': message,
        'priority': 'high'
    }
    
    response = requests.post(url, headers=headers, json=payload)
    
    if response.status_code == 200:
        print('Alert sent:', response.json())
    else:
        print('Error:', response.json())
    
    return response.json()

# Send alert
send_alert('+1-555-123-4567', '[GeoAlert WARNING] Hamilton: Air Quality at 155AQI.')

Rate Limits

  • SMS Sending: 100 messages per hour per user
  • Configuration: 10 requests per minute
  • History Queries: 30 requests per minute

Rate limit info is included in response headers: X-RateLimit-Remaining, X-RateLimit-Reset

Best Practices

  • Always validate phone numbers before calling the API
  • Implement exponential backoff for retries
  • Cache alert configurations to reduce API calls
  • Monitor alert delivery rates and SMS costs
  • Keep messages under 160 characters to avoid multi-part SMS charges
  • Use priority levels to differentiate alert importance
  • Implement duplicate alert prevention (don't spam same alert)
